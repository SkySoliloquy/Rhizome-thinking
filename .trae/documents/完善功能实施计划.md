# Rhizome Thinking 功能完善实施计划

## 项目概述

本项目是一个基于标签连接的个人知识库系统，支持CLI和Web API两种方式使用。目前核心功能包括：节点创建、语义查询、链接管理等。

## 需求分析

### 功能1：用户数据的备份和恢复
- **备份**：将用户的全部数据（节点、元数据）打包导出，**不包含向量数据**
- **恢复**：从备份文件导入数据，恢复到之前的状态，支持热加载
- **前端操作**：通过Web界面进行备份和恢复操作

### 功能2：增强的修改和查询功能
- **多维度查询**：支持时间范围、原始内容关键字、命题关键字等多种查询方式
- **查询模式切换**：与LLM检索查询区分，作为不同的模式开发，使用切换按钮切换
- **节点修改**：对查询到的节点进行字段修改，支持热加载
- **节点删除**：删除查询到的节点，支持热加载

#### 1.1 创建备份管理模块
**文件**: `src/rhizome/core/backup_manager.py`

**功能设计**:
- `backup(output_path: Optional[str])` - 创建备份

### 阶段4：热加载机制实现

#### 4.1 后端热加载支持
**文件**: 
- `src/rhizome/core/node_store.py`
- `src/rhizome/api/routes/nodes.py`

**热加载机制**:
- 所有修改操作（更新、删除）返回操作后的最新数据状态
- API响应包含 `refresh_required: true` 标志
- 可选：使用 WebSocket 推送数据变更事件

#### 4.2 前端热加载实现
**文件**: `src/rhizome/web/static/js/app.js`

**热加载策略**:
```javascript
// 数据管理器
const dataManager = {
    // 修改后自动刷新相关视图
    async updateNode(nodeId, data) {
        const result = await api.updateNode(nodeId, data);
        // 热加载：更新本地数据
        this.refreshNodeInList(nodeId, result.node);
        // 如果当前正在查看该节点，更新详情
        if (windowManager.hasWindow(nodeId)) {
            windowManager.refreshWindow(nodeId);
        }
        ui.showToast('节点已更新');
    },
    
    async deleteNode(nodeId) {
        await api.deleteNode(nodeId);
        // 热加载：从列表移除
        this.removeNodeFromList(nodeId);
        // 关闭相关窗口
        windowManager.closeWindow(nodeId);
        ui.showToast('节点已删除');
    },
    
    async restoreBackup(backupName) {
        await api.restoreBackup(backupName);
        // 热加载：清空并重新加载所有数据
        this.clearCache();
        await this.reloadAllData();
        ui.showToast('备份已恢复，数据已重新加载');
    }
};
```

## 前端热加载机制详解

### 热加载触发时机
1. **修改节点后**：更新该节点在所有视图中的显示
2. **删除节点后**：从所有视图中移除该节点
3. **恢复备份后**：重新加载所有数据并刷新整个页面

### 热加载实现方式
```javascript
// 1. 修改后的热加载
async function handleNodeUpdate(nodeId, newData) {
    // 发送更新请求
    const result = await api.updateNode(nodeId, newData);
    
    // 更新缓存
    nodeCache.set(nodeId, result.node);
    
    // 刷新显示
    refreshNodeDisplay(nodeId, result.node);
}

// 2. 删除后的热加载
async function handleNodeDelete(nodeId) {
    await api.deleteNode(nodeId);
    
    // 从缓存移除
    nodeCache.delete(nodeId);
    
    // 从所有视图移除
    removeNodeFromAllViews(nodeId);
}

// 3. 恢复备份后的热加载
async function handleBackupRestore(backupName) {
    await api.restoreBackup(backupName);
    
    // 清空缓存
    nodeCache.clear();
    
    // 重新加载当前视图
    await reloadCurrentView();
    
    // 显示成功提示
    showToast('备份已恢复，数据已更新');
}
```

## 注意事项

1. **数据安全**
   - 备份操作应保证数据一致性
   - 恢复操作需要用户确认
   - 重要操作前自动创建临时备份

2. **热加载性能**
   - 局部更新优于全局刷新
   - 使用虚拟DOM或增量更新
   - 大数据量时考虑分页加载

3. **兼容性**
   - 备份格式向后兼容
   - 不支持向量数据备份（向量查询已被抛弃）

4. **用户体验**
   - 操作后即时反馈
   - 加载状态提示
   - 错误处理友好提示
   - 中文界面

5. **查询模式区分**
   - 语义检索：基于LLM，返回主题聚合结果
   - 精准查询：基于条件筛选，返回原始节点列表
   - 两种模式通过切换按钮清晰区分
