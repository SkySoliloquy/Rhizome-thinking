# 搜索功能修复总结

## 修复的问题

### 1. 高相关度阈值无结果问题（根本原因已修复）
**问题**：设置 0.8 阈值时几乎找不到内容，0.5 阈值时突然有很多结果

**根本原因**：
- LLM 检索结果被基于排名的固定分数错误过滤
- 排名4+的结果（分数0.75及以下）被 0.8 阈值过滤

**修复**：移除了所有对 LLM 结果的相似度过滤
- `search_optimized.py:129-131`
- `search_stream.py:147-149`
- `themes.py:229-231`

### 2. 新增搜索模式功能（替代相关度滑块）

**三种模式**：

| 模式 | 图标 | 说明 | LLM 行为 |
|-----|------|------|---------|
| 精确 | 🎯 | 只返回高度相关结果 | 返回2-5个最精确的结果 |
| 平衡 | ⚖️ | 平衡精确度和召回率 | 返回5-10个相关结果（默认） |
| 探索 | 🔍 | 发现更多潜在相关内容 | 返回8-15个结果，包括弱关联 |

**修改文件**：
- `llm_search.py`: 根据 search_mode 调整 LLM Prompt
- `search_optimizer.py`: 传递 search_mode 参数
- `index.html`: 用搜索模式按钮替换相似度滑块
- `style.css`: 添加搜索模式按钮样式
- `app.js`: 实现搜索模式切换逻辑
